<task type="auto">
  <name>Task 2: Create SkillsSection.tsx with category-grouped grid and ScrollTrigger.batch stagger</name>
  <files>components/sections/SkillsSection.tsx</files>
  <action>
Create a client component ('use client') following the ProductDemo.tsx animation pattern.

**Imports:**
- useRef from react
- useGSAP from @gsap/react
- gsap from gsap
- ScrollTrigger from gsap/ScrollTrigger
- Card from @/components/ui/Card
- { categories, skillsByCategory } from @/components/sections/skillsData

Register ScrollTrigger: `gsap.registerPlugin(ScrollTrigger)`

**Component structure:**
- `containerRef = useRef<HTMLElement>(null)` for useGSAP scope
- `headingRef = useRef<HTMLDivElement>(null)` for heading fade-in

**useGSAP hook** (scope: containerRef):
Inside `gsap.matchMedia()`:

`"(prefers-reduced-motion: no-preference)"` branch:
1. Heading fade-in: `gsap.from(headingRef.current, { autoAlpha: 0, y: 40, duration: 0.8, ease: 'power2.out', scrollTrigger: { trigger: headingRef.current, start: 'top 85%', toggleActions: 'play none none none' } })`
2. Batch stagger: `ScrollTrigger.batch('.skill-card', { onEnter: (elements) => { gsap.from(elements, { autoAlpha: 0, y: 60, duration: 0.8, stagger: 0.15, ease: 'power2.out' }) }, start: 'top 85%', once: true })`

`"(prefers-reduced-motion: reduce)"` branch:
- `gsap.set('.skill-card', { clearProps: 'all' })`

Return `() => mm.revert()` from the matchMedia setup.

**JSX structure:**
```
<section ref={containerRef} id="skills" className="min-h-screen bg-stone-100 text-stone-900 py-20 sm:py-24 lg:py-32">
  <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">
    <div ref={headingRef} className="text-center mb-16">
      <h2 className="text-4xl sm:text-5xl font-bold mb-4">AI-Powered Skills</h2>
      <p className="text-stone-600 text-lg max-w-2xl mx-auto">Expert-crafted presets that understand your intent and deliver professional results in seconds.</p>
    </div>

    {categories.map(category => (
      <div key={category} className="mb-12 last:mb-0">
        <h3 className="text-burnt-orange uppercase tracking-wide text-sm font-medium mb-6">{category}</h3>
        <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
          {skillsByCategory[category]?.map(skill => (
            <Card key={skill.id} theme="light" hover className="skill-card">
              <div className="flex items-start gap-4">
                <div className="w-12 h-12 rounded-lg bg-burnt-orange/10 flex items-center justify-center flex-shrink-0">
                  <svg viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round" className="w-6 h-6 text-burnt-orange">
                    {skill.iconPaths.map((d, i) => <path key={i} d={d} />)}
                  </svg>
                </div>
                <div>
                  <h4 className="text-lg font-semibold text-stone-900 mb-2">{skill.title}</h4>
                  <p className="text-stone-600 text-sm leading-relaxed">{skill.description}</p>
                </div>
              </div>
            </Card>
          ))}
        </div>
      </div>
    ))}
  </div>
</section>
```

Export as default: `export default function SkillsSection()`

**Important:** Do NOT add `visibility: hidden` or `opacity: 0` via CSS/Tailwind classes on the skill-card elements. GSAP's `autoAlpha` in the `from()` tween handles the initial hidden state. Adding CSS opacity/visibility will conflict with GSAP and cause cards to stay invisible if reduced-motion is enabled.
